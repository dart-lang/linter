Using the Linter

AVOID annotating types for function expression parameters.

Annotating types for function expression parameters is usually unnecessary because the parameter types can almost always be inferred from the context, thus making the practice redundant.

BAD:

var names = people.map((Person person) => person.name);

GOOD:

var names = people.map((person) => person.name);

Incompatible with: always_specify_types.

Usage

To enable the avoid_types_on_closure_parameters lint, add avoid_types_on_closure_parameters under linter > rules in your analysis_options.yaml file:

linter:
  rules:
    - avoid_types_on_closure_parameters
