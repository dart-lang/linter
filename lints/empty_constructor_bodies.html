Using the Linter

From Effective Dart:

DO use ; instead of {} for empty constructor bodies.

In Dart, a constructor with an empty body can be terminated with just a semicolon. This is required for const constructors. For consistency and brevity, other constructors should also do this.

BAD:

class Point {
  int x, y;
  Point(this.x, this.y) {}
}

GOOD:

class Point {
  int x, y;
  Point(this.x, this.y);
}

Usage

To enable the empty_constructor_bodies lint, add empty_constructor_bodies under linter > rules in your analysis_options.yaml file:

linter:
  rules:
    - empty_constructor_bodies
